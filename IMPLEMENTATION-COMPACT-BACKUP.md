# SubConv Next 紧凑管理界面与备份恢复实现

实施日期：2026-07-11
当前发布版本：`1.0.7-31`

## 页面调整

LuCI 菜单统一为四个中文入口：

- 概览：合并服务状态和服务控制，只显示状态、版本、运行时间、监听地址、端口、开机启动及操作按钮。
- 设置：合并网络和数据目录设置，仅保留监听地址、端口、公共访问地址和数据目录。
- 备份与恢复：提供备份导出、分块上传、检查和安全恢复。
- 日志：保留来源、行数、自动刷新、刷新、清空、下载和日志内容，全部中文化并缩小默认高度。

旧 Dashboard、Service、Network、Storage View 和对应样式不再安装。

## 备份格式

格式为 `tar.gz`，`format_version` 当前为 `1`。归档包含：

```text
manifest.json
config/subconv-next
data/...
```

`manifest.json` 记录应用名称、应用版本、创建时间、包含范围，以及每个文件的路径、大小和 SHA-256。

导出时排除日志、缓存、临时文件、PID、锁文件和名称明确包含 secret、token、private-key 的文件。归档文件权限为 `0600`，下载通过 LuCI 登录会话和 ACL 保护的 `cgi-download` 完成。

## 新增接口

读取权限：

- `backup_export`

写入权限：

- `backup_export_cleanup`
- `backup_upload_begin`
- `backup_upload_chunk`
- `backup_check`
- `backup_restore`
- `backup_upload_cancel`

上传由后端生成六位随机会话标识，浏览器不提交服务器目标路径。每块最多 32 KiB，总文件最多 16 MiB。后端按顺序追加并在检查阶段计算归档 SHA-256。

## 恢复安全

- 最大归档 16 MiB、最大解压数据 64 MiB、最多 2000 个文件、单文件最多 16 MiB。
- 只接受普通文件，拒绝目录项、符号链接、硬链接、设备、套接字和 FIFO。
- 拒绝绝对路径、反斜杠路径、非规范路径、`..`、重复路径和未知顶层路径。
- 检查 `manifest.json` 字段、格式版本、文件集合、大小和 SHA-256。
- 恢复目标只能是当前 UCI 配置中经过白名单与真实路径检查的 `data_dir`。
- 恢复时保留当前设备的数据目录位置，不信任归档中的目标路径、属主和权限。
- 使用独占恢复锁阻止并发恢复。
- 在目标文件系统检查可用空间，先准备新目录，再停止服务并原子替换。
- 当前配置和数据以同文件系统回滚副本保留；替换、权限设置或服务重启失败时自动恢复原数据。
- 普通文件恢复为 `0600`，目录为 `0700`，UCI 配置为 `0644`。

## 远程自定义规则可靠性

从 `1.0.7-25` 开始，HTTP 类型的 YAML/TEXT 自定义规则在生成阶段由 SubConv Next 下载、校验并转换为 inline provider。最终订阅不再依赖 Mihomo 启动后临时下载远程规则。

- 远程下载成功后按 URL 缓存规则快照。
- 后续下载失败时继续使用上一次有效快照。
- 首次下载失败且没有缓存时终止本次刷新，保留现有有效 YAML。
- 空 payload 或损坏 YAML 会被拒绝，不生成缺少自定义规则的配置。
- 单个远程规则文件限制为 2 MiB。
- MRS 是二进制格式，当前仍保留为 Mihomo 运行时 provider，并记录警告。

自定义规则编辑器会检测复制粘贴时缺少换行导致的连续规则，例如 `DOMAIN-SUFFIX,a.comDOMAIN-SUFFIX,b.com`，自动拆分后再去重。统计信息分别显示物理输入行、修复的粘连位置、无效或注释行和重复规则；后端配置规范化执行相同修复，避免绕过前端后写入畸形 payload。

“新建同名代理组”模式只使用规则的图标和显示名称生成组名，不再读取隐藏的现有目标字段。旧版本产生的 `new_group + 🚀 节点选择` 组合会自动迁移，避免生成 `🚀 节点选择 2` 等冲突名称。

## 验证结果

- `go test ./...` 通过。
- 覆盖导出、清单、日志/缓存排除、成功恢复、损坏归档、版本不兼容、校验失败、绝对路径、`..`、符号链接、硬链接、恢复锁和服务启动失败回滚。

## 环境限制

已在 `10.0.0.1` 的 Kwrt 25.12.5、LuCI Material3 主题上安装 `1.0.7-23` 并完成真实设备验收。补充验收确认顶部导航与页面标题、主体使用同一公共容器，页面切换不发生横向跳动；概览页“打开 WebUI”按钮使用 RPC 返回并验证过的地址，不读取或拼接未保存的前端输入。

```text
artifacts/luci-1.0.7-23/desktop-overview.png
artifacts/luci-1.0.7-23/mobile-overview.png
```

设备主题自身会请求一个不存在的 `view/status/include/50_dsl.js` 并返回 404；该请求与 SubConv Next 页面无关，未影响页面加载或交互。

备份包含迁移所必需的业务配置和订阅数据，归档未加密。虽然下载受 LuCI 登录与 ACL 保护，用户仍应将导出的备份视为敏感文件并妥善保存。
